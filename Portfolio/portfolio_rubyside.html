RubySide

The RubySide project was set by a company called Cleaversteam where they asked for their RubySide website to be redesigned to include the latest modern interface trends.

Research

The first piece of research carried out was to look at the current website to collect the logo and the brand colours. Once this was done the next step was to research current trends and view other website related to RubySide. This was then all brought together to be able to gather ideas for design stage.

Design

There were two design stages, the first was the mobile design and the second was the desktop design. At both stages of design, they would first be sketched out on paper and pencil. The design was then moved on to the computer by coding the design before functionality was added to the development stage. The extra design stage which was included while design the desktop was to create SVG of rubies to be included in the design. Once all the design stage was complete the next step was to collect the content ready for the functionality to be added to the website.

Development

At this stage, the design had been created in code so the website was ready for the functionality to be added. The process of moving the design from paper to code always starts by creating an SCSS/CSS file and an Html file and then working from the header to the footer. The functionality which was adding links and making sure the website can be responsive to the desktop and mobile designs. Once the functionality was added the website was ready to be tested.

Test

This project was tested through manual testing to make sure all functionality works, for example, do the links got to the correct part of the page or are the social media links working.
